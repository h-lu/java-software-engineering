# 全书贯穿项目：CampusFlow（团队项目）

## 项目概述

CampusFlow 是一个开放式的项目框架——每 2-3 人团队选择一个具体场景，实现一个真实可用的 Web 应用。

**项目性质**：不是"假项目"，而是"能用的小工具"

**核心理念**：
- AI 是倍增器，不是替代者
- 架构决策必须由团队做（通过 ADR 记录）
- 首席架构师轮换，确保概念完整性

---

## 技术要求

### 后端
- Java 17+
- **Javalin**（极简 Web 框架）- 保持极简，不引入 Spring Boot
- SQLite（轻量级数据库）
- JDBC

### 前端
- **AI 生成**：HTML + CSS + JavaScript
- **人审查**：使用 AI 审查检查清单确保质量
- 不追求美观，追求可用

### 工具链
- Maven/Gradle
- Git + GitHub
- JUnit 5
- SpotBugs/PMD

---

## 关键机制

### 1. ADR（架构决策记录）

**每两周一篇，共 4 篇**：

| ADR | 提交周 | 内容 |
|-----|--------|------|
| ADR-001 | Week 02 | 领域模型设计决策 |
| ADR-002 | Week 04 | 数据存储方案决策 |
| ADR-003 | Week 06 | API 设计决策 |
| ADR-004 | Week 08 | 架构演进决策 |

**首席架构师负责撰写**，团队参与讨论

### 2. 首席架构师轮换

**轮换周期**：每两周一次

| 轮次 | 周次 | 职责 |
|------|------|------|
| 第1轮 | Week 02-03 | 设计领域模型，写 ADR-001 |
| 第2轮 | Week 04-05 | 设计数据层，写 ADR-002 |
| 第3轮 | Week 06-07 | 设计 API，写 ADR-003 |
| 第4轮 | Week 08-09 | 架构演进，写 ADR-004 |

**职责**：
- 主持设计讨论
- 撰写当期 ADR
- 做最终设计决策（需听取团队意见）

## 选题池

### 任务管理类

**1. TaskFlow（个人任务管理）**
- 功能：创建任务、设置截止日期、标记完成、分类标签
- 差异化点：时间线视图、优先级算法

**2. TeamSync（团队协作）**
- 功能：团队创建、任务分配、进度跟踪、评论
- 差异化点：看板视图、通知机制

**3. HabitForge（习惯养成）**
- 功能：习惯创建、打卡记录、连续天数统计
- 差异化点：可视化图表、习惯模板

### 知识管理类

**4. NoteStream（笔记工具）**
- 功能：创建笔记、标签分类、搜索、导出
- 差异化点：Markdown 支持、双向链接

**5. CodeVault（代码片段管理）**
- 功能：保存代码片段、语法高亮、标签、搜索
- 差异化点：代码执行、共享链接

### 资源管理类

**6. BookHub（图书管理）**
- 功能：图书录入、借阅记录、归还提醒
- 差异化点：扫码录入、推荐算法

**7. DeviceLend（设备借用）**
- 功能：设备登记、借用申请、归还确认
- 差异化点：冲突检测、排队机制

**8. RoomReserve（会议室预约）**
- 功能：查看空闲、预约、取消、冲突检测
- 差异化点：日历视图、重复预约

### 生活工具类

**9. SpendWise（账单分析）**
- 功能：记录支出、分类统计、月度报告
- 差异化点：预算提醒、数据可视化

**10. HealthTrack（健康追踪）**
- 功能：记录体重、运动、睡眠
- 差异化点：趋势分析、目标设定

## 差异化要求

**相同选题必须有差异化设计**：

| 维度 | 差异化方向 |
|------|-----------|
| 功能集 | 核心功能相同，但附加功能不同 |
| 交互方式 | 命令行 vs Web vs 移动端 |
| 数据模型 | 不同的实体关系设计 |
| 技术亮点 | 不同的技术特色（如搜索、推荐、可视化）|

## Week 10 AI 审查训练（重点）

这是课程的核心周，学生需要：

1. **用 AI 生成前端代码**
2. **使用审查检查清单评估**：
   - 功能正确性
   - 边界情况（AI 常遗漏）
   - 安全考虑（SQL 注入、XSS）
   - 可维护性（命名、幻觉）
3. **修复并改进**
4. **提交审查报告**

**目标**：不只是用 AI，而是学会判断 AI 输出的质量
