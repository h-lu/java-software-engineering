# 目录

## 如何使用本书

这本书不是字典——请**按顺序读**。每周的内容都建立在上一周的基础上，跳过任何一周都可能让你在后面卡住。

每周你会做两件事：一个**当周实验/练习**（当周完结）和**CampusFlow 进度**（贯穿 16 周的团队项目）。前者让你当周就有“我真的写出来了”的成就感，后者让你在学期末拿到一个能演示、能复盘、能写进简历的工程作品。

书中有三个经常出场的角色：零基础的**小北**（就是你）、爱问问题的**阿码**、和工作多年的**老潘**。他们会在你困惑的时候帮你提问、在你得意的时候给你泼冷水。

## 阶段一：思维奠基（Week 01-04）

> 从“写代码”到“做工程”。前四周不追求堆技术栈，追求把思维打稳：抽象、设计、错误处理、协作流程。

| 周次 | 章节 | 当周实验/练习 | CampusFlow 进度 |
|------|------|-------------|--------------|
| 01 | [抽象层次与软件工程史](week_01/CHAPTER.md) | Java 环境 + 第一个可运行程序 | 项目立项与仓库初始化 |
| 02 | [类设计与概念完整性](week_02/CHAPTER.md) | 领域模型草图与类设计练习 |
| 03 | [异常处理与防御式编程](week_03/CHAPTER.md) | 让程序“不崩”的输入处理 | 失败模式清单 |
| 04 | [团队协作与 Code Review](week_04/CHAPTER.md) | PR 流程与审查训练 | ADR-002（数据存储） |

## 阶段二：系统化工程（Week 05-08）

> 从“能跑”到“靠谱”。你会开始建立质量门禁、理解数据完整性、学习模式与重构的取舍。

| 周次 | 章节 | 当周实验/练习 | CampusFlow 进度 |
|------|------|-------------|--------------|
| 05 | [集合框架与抽象选择](week_05/CHAPTER.md) | List/Map/Stream 的选择题 | 数据结构选型记录 |
| 06 | [测试策略与质量门禁](week_06/CHAPTER.md) | JUnit 5 测试矩阵 | ADR-003（API 设计） |
| 07 | [持久化与数据完整性](week_07/CHAPTER.md) | JDBC + Repository 入门 | SQLite 持久化落地 |
| 08 | [设计模式与重构决策](week_08/CHAPTER.md) | 用模式改一次“坏味道” | ADR-004（架构演进） |

## 阶段三：AI 时代的工程（Week 09-12）

> 你会把“AI 是倍增器”的理念落到可操作的流程：生成、审查、修正、门禁；并引入静态分析与质量指标。

| 周次 | 章节 | 当周实验/练习 | CampusFlow 进度 |
|------|------|-------------|--------------|
| 09 | [REST API 设计](week_09/CHAPTER.md) | 用 Javalin 交付一组接口 | API 契约与端到端验收 |
| 10 | [AI 辅助开发与审查（核心周）](week_10/CHAPTER.md) | 审查 AI 生成代码训练 | 审查清单落地 |
| 11 | [质量保障与静态分析](week_11/CHAPTER.md) | SpotBugs/PMD/JaCoCo 入门 | 质量门禁上线 |
| 12 | [技术债与架构演进](week_12/CHAPTER.md) | 技术债清单与重构计划 | 演进式架构复盘 |

> 最后四周主线只有一个：交付。你会经历集成、Bug Bash、文档沉淀、发布准备与最终复盘。

| 周次 | 章节 | 当周实验/练习 | CampusFlow 进度 |
|------|------|-------------|--------------|
| 13 | [集成、测试与 Bug Bash](week_13/CHAPTER.md) | 跨组互测与根因分析 | 集成测试覆盖 |
