# 目录

## 如何使用本书

这本书不是字典——请**按顺序读**。每周的内容都建立在上一周的基础上，跳过任何一周都可能让你在后面卡住。

每周你会做两件事：一个**当周实验/练习**（当周完结）和**CampusFlow 进度**（贯穿 16 周的团队项目）。前者让你当周就有"我真的写出来了"的成就感，后者让你在学期末拿到一个能演示、能复盘、能写进简历的工程作品。

书中有三个经常出场的角色：零基础的**小北**（就是你）、爱问问题的**阿码**、和工作多年的**老潘**。他们会在你困惑的时候帮你提问、在你得意的时候给你泼冷水。

## 阶段一：思维奠基（Week 01-04）

> 从"写代码"到"做工程"。前四周不追求堆技术栈，追求把思维打稳：抽象、设计、错误处理、协作流程。

| 周次 | 章节 | 当周实验/练习 | CampusFlow 进度 |
|------|------|-------------|--------------|
| 01 | [从 Python 到 Java——抽象层次与软件工程史](week_01/CHAPTER.md) | Java 环境 + 第一个可运行程序 | 项目立项与仓库初始化 |
| 02 | [类设计与领域建模](week_02/CHAPTER.md) | 领域模型草图与类设计练习 | ADR-001（领域模型） |
| 03 | [异常处理与防御式编程](week_03/CHAPTER.md) | 让程序"不崩"的输入处理 | 失败模式清单 |
| 04 | [团队协作与代码审查](week_04/CHAPTER.md) | PR 流程与审查训练 | ADR-002（数据存储） |

## 阶段二：系统化工程（Week 05-08）

> 从"能跑"到"靠谱"。你会开始建立质量门禁、理解数据完整性、学习模式与重构的取舍。

| 周次 | 章节 | 当周实验/练习 | CampusFlow 进度 |
|------|------|-------------|--------------|
| 05 | [集合框架与数据结构选择](week_05/CHAPTER.md) | List/Map/泛型的选择题 | 数据结构选型记录 |
| 06 | [单元测试与测试策略](week_06/CHAPTER.md) | JUnit 5 测试矩阵 | 测试覆盖率达到 80%+ |
| 07 | [数据持久化与数据库访问](week_07/CHAPTER.md) | JDBC + Repository 入门 | SQLite 持久化落地 |
| 08 | [重构、设计模式与代码质量](week_08/CHAPTER.md) | 用模式改一次"坏味道" | ADR-003（架构演进） |

| 周次 | 章节 | 当周实验/练习 | CampusFlow 进度 |
|------|------|-------------|--------------|
