🐍 ➡️ ☕ Python 到 Java 的第一次碰撞

小北：我都还没运行，它怎么就报错？Python 里不都这么写吗？

Python（✅ 正常）

name = "张三"
job = "软件工程师"
print(name)

Java（❌ 编译错误）

name = "张三";  // Error!
job = "软件工程师";
// cannot find symbol: variable name

老潘：这是静态类型和动态类型的第一次正面碰撞。

🔍 静态类型 vs 动态类型

维度 Python（动态） Java（静态）
变量声明 name = "张三" String name = "张三";
类型检查 运行时检查 编译期检查 ✅
灵活性 高（随时换类型） 低（声明后不变）
安全性 低（运行时才报错） 高（编译期发现错误）
适合场景 快速原型、脚本 大型项目、团队协作

✅ 正确的 Java 写法

// 文件：VariableDemo.java
public class VariableDemo {
    public static void main(String[] args) {
        // 必须先声明类型
        String name = "张三";
        String job = "软件工程师";
        String email = "zhangsan@example.com";
        int age = 28;
        double yearsOfExp = 5.5;

        System.out.println("姓名：" + name);
        System.out.println("职位：" + job);
    }
}

老潘：这不是"麻烦"，这是契约——你和编译器签了合同，它会帮你提前发现问题。

💬 让程序"活"起来：Scanner

阿码：能不能像 Python 的 input() 那样运行时问用户要信息？

import java.util.Scanner;  // 必须导入

public class ScannerDemo {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.print("请输入名字：");
        String name = scanner.nextLine();

        System.out.print("请输入年龄：");
        int age = Integer.parseInt(scanner.nextLine());

        System.out.println("姓名：" + name + "，年龄：" + age);
        scanner.close();
    }
}

📋 Scanner 常用方法

方法 读取内容 示例输入
nextLine() 一整行字符串 ⭐推荐 "Hello World"
next() 下一个单词（空格分隔） "Hello"
nextInt() 整数（⚠️ 有换行符问题） 123
nextDouble() 浮点数 3.14

小北踩坑：nextInt() 后面直接用 nextLine() 会跳过输入！

⚠️ 小北踩过的坑：nextInt() 遗留换行符

❌ 有问题的代码

System.out.print("年龄：");
int age = scanner.nextInt();  // 读取 25

System.out.print("姓名：");
String name = scanner.nextLine();  // 直接跳过！

// 输出：姓名：，年龄：25

✅ 修复方法

// 推荐：统一用 nextLine()
System.out.print("年龄：");
int age = Integer.parseInt(
    scanner.nextLine()
);

System.out.print("姓名：");
String name = scanner.nextLine();  // 正常工作

老潘：nextInt() 只读取数字，不读取换行符。\n 留在缓冲区，被 nextLine() 当成"空行"。

💼 完整的名片生成器（交互版）

import java.util.Scanner;

public class BusinessCardInteractive {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);

        System.out.print("请输入姓名：");
        String name = scanner.nextLine();

        System.out.print("请输入职位：");
        String job = scanner.nextLine();

        System.out.print("请输入年龄：");
        int age = Integer.parseInt(scanner.nextLine());

        // 格式化输出
        System.out.println("================================");
        System.out.println("       " + name + " 的个人名片");
        System.out.println("================================");
        System.out.println("职位：" + job);
        System.out.println("年龄：" + age + "岁");
        System.out.println("================================");

        scanner.close();
    }
}

🛡️ Python 的宽容 vs Java 的严格

小北：Java 太麻烦了！Python 两行搞定的事情，Java 要写十行。

老潘：来，我给你看个场景。

Python 问题

age = input("年龄：")  # "25"
result = age + "10"    # 得到 "2510"
# 静默拼接，bug 很隐蔽！

Java 保护

String age = scanner.nextLine();
int result = age + 10;  // ❌ 编译错误！
// bad operand types
// 必须先 parseInt()

⏱️ 编译期检查 vs 运行时检查

🐍 Python

编写 → 运行 → 💥 报错

错误到运行时才暴露
可能已在生产环境

VS

☕ Java

编写 → 🛡️ 编译报错 → 运行

错误在编译期就被拦截
代码根本跑不起来

老潘：你宁可多写几个字符的类型声明，也不愿意半夜三点被电话叫起来修复线上 bug。

🎨 printf 格式化输出

System.out.println("╔═══════════════════════════════════╗");
System.out.printf("║      %s 的个人名片%n", name);
System.out.println("╠═══════════════════════════════════╣");
System.out.printf("║ 职位：%-20s ║%n", job);      // 左对齐
System.out.printf("║ 年龄：%d 岁%15s ║%n", age, ""); // %d 整数
System.out.printf("║ 经验：%.1f 年%12s ║%n", exp, ""); // %.1f 一位小数
System.out.println("╚═══════════════════════════════════╝");
%s 字符串 | %d 整数 | %.1f 浮点数
%-20s 左对齐占20字符 | %n 换行符

💬 注释应该解释"为什么"

❌ 过度注释

// 创建一个 Scanner 对象
Scanner scanner = new Scanner(System.in);
// 打印提示信息
System.out.print("请输入姓名：");
// 读取用户输入的姓名
String name = scanner.nextLine();

注释翻译了代码，没有价值

✅ 好的注释

// 使用 nextLine() 而非 nextInt()，
// 避免 nextInt() 遗留换行符问题
int age = Integer.parseInt(scanner.nextLine());

// 使用中文全角边框以匹配
// 中文字符显示宽度
System.out.println("╔═══════╗");

解释"为什么这样做"
