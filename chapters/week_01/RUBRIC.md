# Week 01 评分标准

> **总分**：100 分 + 加分项 10 分
> **评分周期**：Week 02 上课前

---

## 评分维度

| 维度 | 占比 | 说明 |
|------|------|------|
| 基础练习 | 30% | 环境配置、Hello World、变量声明、Scanner 输入 |
| 进阶练习 | 30% | 完整名片生成器、类型安全练习 |
| 代码质量 | 20% | 命名规范、代码结构、注释 |
| 团队项目 | 20% | 团队组建、选题确定、Git 仓库、项目锚点 |
| AI 练习 | 加分 10% | AI 协作练习（可选） |

## 1. 基础练习（30 分）

### 1.1 环境配置验证（8 分）

| 评分项 | 分数 | 验收标准 |
|--------|------|---------|
| JDK 安装 | 2 分 | `java -version` 输出包含 "17.x.x" |
| Maven 安装 | 2 分 | `mvn -v` 输出正确版本信息 |
| Git 配置 | 2 分 | `git config --list` 包含 user.name 和 user.email |
| IntelliJ 配置 | 2 分 | 能在 IntelliJ 中创建并运行 Java 项目 |

**评分说明**：
- A（7-8 分）：全部完成，有截图证明
- B（5-6 分）：完成 3 项，或有小问题但不影响使用
- C（3-4 分）：完成 2 项，有明显问题
- D（0-2 分）：完成少于 2 项，或无法正常工作

---

### 1.2 Hello World 程序（6 分）

| 评分项 | 分数 | 验收标准 |
|--------|------|---------|
| 代码结构 | 2 分 | 类名、main 方法签名正确 |
| 编译通过 | 2 分 | 代码能成功编译 |
| 输出正确 | 2 分 | 运行后输出与示例一致 |

**评分说明**：
- A（6 分）：完全符合要求，代码整洁
- B（4-5 分）：能运行，有小瑕疵（如输出格式略有偏差）
- C（2-3 分）：能编译，但有明显错误（如 main 方法签名错误）
- D（0-1 分）：无法编译或无法运行

---

### 1.3 变量声明练习（8 分）

| 评分项 | 分数 | 验收标准 |
|--------|------|---------|
| 类型声明 | 3 分 | 正确使用 `String`、`int`、`double` 等类型 |
| 变量命名 | 2 分 | 变量名有意义，不是 `x`、`s1` 等 |
| 字符串拼接 | 2 分 | 正确使用 `+` 拼接字符串 |
| 理解输出 | 1 分 | 能用自己的话解释静态类型 |

**评分说明**：
- A（7-8 分）：全部正确，命名规范，理解清晰
- B（5-6 分）：基本正确，命名有待改进
- C（3-4 分）：能用但不规范（如用 `s1`、`s2` 命名）
- D（0-2 分）：有明显错误（如类型不匹配）

---

### 1.4 Scanner 输入练习（8 分）

| 评分项 | 分数 | 验收标准 |
|--------|------|---------|
| 导入 Scanner | 1 分 | 正确导入 `java.util.Scanner` |
| 创建 Scanner | 2 分 | 正确创建 `Scanner(System.in)` |
| 读取输入 | 3 分 | 正确使用 `nextLine()` 和 `Integer.parseInt()` |
| 资源管理 | 1 分 | 记得关闭 Scanner |
| 输出格式 | 1 分 | 输出格式正确 |

**评分说明**：
- A（7-8 分）：完全符合要求，处理了换行符问题
- B（5-6 分）：基本正确，可能有小问题（如直接用 `nextInt()` 但不影响功能）
- C（3-4 分）：能读取输入，但有明显缺陷（如换行符导致后续读取跳过）
- D（0-2 分）：无法正常读取输入

## 2. 进阶练习（30 分）

### 2.2 类型安全练习（15 分）

| 评分项 | 分数 | 验收标准 |
|--------|------|---------|
| 代码对比 | 5 分 | 正确实现两种版本（String vs int） |
| 思考题 1 | 3 分 | 理解为什么版本 1 编译不通过 |
| 思考题 2 | 3 分 | 能对比 Python 和 Java 的行为 |
| 思考题 3 | 4 分 | 能结合电商场景说明类型安全的价值 |

**思考题评分标准**：

**思考题 1**：为什么版本 1 编译不通过？
- A（3 分）：清晰解释"类型不匹配"，理解这是编译期保护
- B（2 分）：能说出"类型不对"，但解释不够深入
- C（1 分）：只说"Java 不允许"，不理解原因

**思考题 2**：Python 的行为哪个更安全？
- A（3 分）：能解释 Python 会"静默失败"（字符串拼接），Java 在编译期拦截
- B（2 分）：能说出区别，但不够准确
- C（1 分）：只说"Python 更灵活"

**思考题 3**：电商场景的后果
- A（4 分）：能给出具体场景（如价格计算错误），说明影响
- B（2-3 分）：能说明后果，但场景不够具体
- C（1 分）：只说"会出 bug"

## 3. 代码质量（20 分）

### 3.1 命名规范（8 分）

| 评分项 | 分数 | 验收标准 |
|--------|------|---------|
| 类名 | 2 分 | PascalCase，如 `BusinessCard` |
| 变量名 | 3 分 | camelCase，有意义，如 `userName`、`cardWidth` |
| 常量 | 1 分 | 全大写下划线，如 `MAX_SIZE`（如果有） |
| 一致性 | 2 分 | 命名风格前后一致 |

**评分说明**：
- A（7-8 分）：完全符合 Java 命名规范
- B（5-6 分）：基本符合，偶有不一致
- C（3-4 分）：有明显问题（如 `s1`、`x`、`temp`）
- D（0-2 分）：命名混乱，无法理解

---

### 3.2 代码结构（7 分）

| 评分项 | 分数 | 验收标准 |
|--------|------|---------|
| 缩进正确 | 2 分 | 使用 4 个空格缩进 |
| 大括号风格 | 2 分 | K&R 风格（左大括号不换行） |
| 行长度 | 1 分 | 每行不超过 100 字符 |
| 空行使用 | 2 分 | 逻辑段落之间有空行 |

**评分说明**：
- A（6-7 分）：结构清晰，易读
- B（4-5 分）：基本规范，有小瑕疵
- C（2-3 分）：缩进混乱，大括号风格不一致
- D（0-1 分）：无法阅读

---

### 3.3 注释（5 分）

| 评分项 | 分数 | 验收标准 |
|--------|------|---------|
| 类注释 | 2 分 | 类有 Javadoc 注释（简要说明） |
| 关键逻辑 | 2 分 | 关键代码有行内注释 |
| 注释质量 | 1 分 | 注释解释"为什么"而非"是什么" |

**评分说明**：
- A（5 分）：注释恰当，解释关键逻辑
- B（3-4 分）：有基本注释，但质量一般
- C（1-2 分）：注释不足或过度注释（翻译代码）
- D（0 分）：无注释

## 总分计算

### 必做部分（100 分）

```
总分 = 基础练习(30) + 进阶练习(30) + 代码质量(20) + 团队项目(20)
```

### 加分项（10 分）

```
最终得分 = 总分 + AI 练习(0-10) + 挑战练习(0-2)
```

**注意**：总分最高不超过 100 分（加分项用于提升薄弱环节）。

## 验收命令

```bash
java -version
mvn -v
git config --list
```

```bash
cd /path/to/your/project
javac HelloWorld.java
java HelloWorld
```

### Git 验证
```bash
git log --oneline
git status
```

## 常见扣分项

### 轻微问题（-1 分）
- 输出格式略有偏差
- 变量命名不够规范
- 注释不足

### 中等问题（-2-3 分）
- 代码结构混乱
- 命名难以理解
- 缺少关键功能

### 严重问题（-5 分以上）
- 无法编译或运行
- 明显抄袭（直接复制 starter_code）
- 团队项目未完成
