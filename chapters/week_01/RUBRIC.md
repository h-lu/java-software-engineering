# Week 01 评分标准（Rubric）

## 评分维度

### 1. 开发环境配置（20 分）

| 等级 | 分数 | 描述 |
|------|------|------|
| 优秀 | 18-20 | JDK 17、IntelliJ IDEA、Maven 全部正确安装，版本验证通过 |
| 良好 | 15-17 | 环境基本配置完成，有小的配置问题但不影响使用 |
| 合格 | 12-14 | 环境配置完成，但存在明显问题（如版本不对） |
| 不合格 | 0-11 | 环境配置不完整，无法运行 Java 程序 |

**检查点**：
- [ ] `java -version` 显示 17.x
- [ ] `mvn -version` 正常显示
- [ ] IntelliJ IDEA 可正常启动
- [ ] Git 用户信息已配置

---

### 2. Hello World 程序（30 分）

| 等级 | 分数 | 描述 |
|------|------|------|
| 优秀 | 27-30 | 程序正确运行，包含多个类型变量，代码规范 |
| 良好 | 22-26 | 程序正确运行，但变量类型不够多样或代码有小问题 |
| 合格 | 18-21 | 程序基本正确，有轻微错误 |
| 不合格 | 0-17 | 程序无法编译或运行 |

**检查点**：
- [ ] 能正确输出 "Hello, World!"
- [ ] 声明了至少 3 个不同类型的变量
- [ ] 代码格式规范（缩进、命名）
- [ ] 进阶：使用 Scanner 获取输入（加分项）

---

### 3. 个人信息卡片程序（30 分）

| 等级 | 分数 | 描述 |
|------|------|------|
| 优秀 | 27-30 | 功能完整，交互流畅，处理了换行符问题，输出格式美观 |
| 良好 | 22-26 | 功能完整，但有小问题（如换行符处理不当） |
| 合格 | 18-21 | 基本功能实现，有明显缺陷 |
| 不合格 | 0-17 | 程序无法运行或功能不完整 |

**检查点**：
- [ ] 使用 Scanner 获取用户输入
- [ ] 正确处理 `nextInt()` 后的换行符
- [ ] 输出格式整齐美观
- [ ] 程序结束时关闭 Scanner
- [ ] 代码有适当注释

---

### 4. 团队组建与项目启动（20 分）

| 等级 | 分数 | 描述 |
|------|------|------|
| 优秀 | 18-20 | 团队组建完成，GitHub 仓库建立，有清晰的 README |
| 良好 | 15-17 | 团队组建完成，仓库建立，但 README 不够完整 |
| 合格 | 12-14 | 团队组建完成，但仓库设置有问题 |
| 不合格 | 0-11 | 未完成团队组建或仓库未建立 |

**检查点**：
- [ ] 团队成员名单明确
- [ ] 项目选题确定
- [ ] GitHub 仓库可访问
- [ ] 第一轮 Chief Architect 确定
- [ ] 仓库有基本的 .gitignore 配置

---

## 总分计算

| 维度 | 权重 | 满分 |
|------|------|------|
| 环境配置 | 20% | 20 |
| Hello World | 30% | 30 |
| 个人信息卡片 | 30% | 30 |
| 团队组建 | 20% | 20 |
| **总分** | **100%** | **100** |

---

## 评分流程

1. **自动检查**（30%）
   - 代码是否能编译
   - 基本功能是否通过

2. **人工评审**（70%）
   - 代码质量
   - 格式规范
   - 文档完整性

---

## 迟交政策

- 迟交 1 天内：扣 10%
- 迟交 2 天内：扣 25%
- 迟交 3 天及以上：扣 50%

---

## 学术诚信

- 允许讨论思路，但代码必须自己编写
- 禁止直接复制他人代码（包括 AI 生成的代码不加修改地提交）
- 使用 AI 辅助必须记录使用日志
