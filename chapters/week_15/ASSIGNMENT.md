# Week 15 作业：项目集市与展示准备

## 作业说明

本周不涉及新的 Java 代码开发，重点是**为项目集市做准备**。你需要完成以下任务，让你的 CampusFlow 展示更加专业和自信。

---

## 基础作业（必做，100 分满分）

### 任务 1：设计演示脚本（30 分）

为你的 CampusFlow 项目设计一个 8-10 分钟的演示脚本。

**要求**：
1. 遵循"开场 1 分钟 + 主体 7 分钟 + 结尾 2 分钟"的时间分配
2. 包含以下五个部分：
   - 问题背景（为什么做这个项目）
   - 技术选型与架构（Java + Javalin + SQLite）
   - 功能演示（核心功能）
   - 工程实践（测试、文档、部署）
   - 踩坑与成长（Bug Bash、AI 协作经验）
3. 选择 3-4 个亮点详细讲，其他内容一句话带过
4. 准备 10-15 个常见问题的回答（见任务 3）

**提交**：
- 提交文件：`presentation_script.md`
- 格式：Markdown，包含每个部分的关键点（不是完整逐字稿）

---

### 任务 2：准备展示材料（40 分）

设计你的展示 PPT 和海报。

**PPT 要求**：
1. 页数：12-15 页（不含封面和 Q&A）
2. 设计原则：
   - 一页一信息（不要堆砌多个概念）
   - 文字精简（用关键词，不是完整句子）
   - 图胜于文（用架构图、截图、表格，而不是纯文字）
   - 配色统一（不超过 3 种主色）
3. 必须包含：
   - 封面（项目名、团队、一句话介绍）
   - 目录
   - 问题背景（现有工具 vs 你的方案）
   - 技术选型（技术栈表格 + 架构图）
   - 功能演示（核心功能列表 + 截图/录屏）
   - 工程实践（测试覆盖率图表 + CI/CD 截图）
   - 踩坑与成长（Bug Bash 表格 + AI 协作经验表格）
   - 项目链接（GitHub 二维码、在线演示二维码）
   - Q&A 页

**海报要求**：
1. 尺寸：A1（594 × 841 mm）
2. 设计原则：
   - 远距离可读（3 米外能看清标题）
   - 层级清晰（标题 → 核心内容 → 技术栈 → 亮点 → 二维码）
   - 字体大小：标题 ≥ 72pt，正文 ≥ 24pt
3. 必须包含：
   - 项目标题和一句话介绍
   - 问题 vs 解决方案对比
   - 技术栈列表
   - 核心亮点（3-4 个）
   - GitHub 和在线演示的二维码

**提交**：
- 提交文件：`slides.pptx` / `poster.pdf`
- 或提交设计源文件：`slides.html` / `poster.html`

---

### 任务 3：准备问答应对（30 分）

准备 10-15 个常见问题的答案。

**必须准备的问题类型**：
1. 技术选型类：
   - 为什么用 Java？不用 Python 或 Go？
   - 为什么不用 Spring？
   - 为什么用 SQLite？不用 MySQL 或 PostgreSQL？

2. 设计决策类：
   - 你们的架构有什么特色？
   - 你们的 API 设计遵循 RESTful 吗？
   - 你们的测试覆盖率为什么是 80%？

3. 工程实践类：
   - 你们的 CI/CD 流水线是什么样的？
   - 你们用了 AI 工具吗？效果如何？
   - Bug Bash 发现了什么问题？

4. 对比与竞争类：
   - CampusFlow 和 Notion / Todoist 相比有什么优势？
   - 你们的创新点在哪里？

5. 未来规划类：
   - 下一步有什么计划？
   - 你们的项目能商业化吗？

**回答要求**：
- 每个回答包含三部分：直接回答 + 解释原因 + 补充说明
- 回答控制在 1-2 分钟（不要啰嗦）
- 准备"诚实回答"策略：如果被问到没想到的问题，说"这个问题我们确实没有考虑过，感谢您的建议"

**提交**：
- 提交文件：`qa_prep.md`
- 格式：Markdown，包含问题和答案

## 进阶作业（可选，额外加分）

### 任务 4：跨组展示演练（+20 分）

邀请 1-2 个其他团队观看你的展示演练，并收集反馈。

**要求**：
1. 进行完整的 8-10 分钟演示
2. 邀请观看者填写反馈表（见 `examples/15_feedback_form.md`）
3. 根据反馈调整你的脚本、材料、表达

**提交**：
- 提交文件：`showcase_practice.md`
- 包含：演练时间、观看者反馈、你的调整计划

---

### 任务 5：AI 协作练习（+15 分）

使用 AI 工具（如 ChatGPT、Gamma、Beautiful.ai）生成 PPT 草稿，然后进行人工审查和优化。

**要求**：
1. 使用 AI 生成 PPT 草稿（输入你的演示大纲）
2. 使用"前端审查检查清单"评估 AI 生成的内容：
   - 功能正确性：PPT 是否准确反映了你的内容？
   - 边界情况：AI 是否遗漏了重要内容？
   - 安全考虑：无（PPT 不涉及安全问题）
   - 可维护性：是否容易修改和更新？
   - AI 特定问题：是否有幻觉？（不存在的功能、错误的技术术语）
3. 记录审查发现的问题，并进行人工优化
4. 提交审查报告

**提交**：
- 提交文件：`ai_ppt_review.md`
- 包含：AI 生成的截图、审查发现的问题、优化后的版本、反思

## 常见问题

### Q1：我没有设计经验，怎么做 PPT？
A1：
1. 使用提供的 `examples/15_showcase_poster.html` 作为模板
2. 使用 AI 工具（如 Gamma）生成草稿，然后人工优化
3. 参考本周第 3 节的 PPT 设计原则

### Q2：我真的很紧张，怕忘词怎么办？
A2：
1. 不要逐字背稿，记住结构（开场 → 背景 → 选型 → 演示 → 实践 → 成长 → 结尾）
2. 练习 5-10 次，从"背词"到"讲故事"
3. 准备 cue card（只写关键词，不是完整句子）
4. 邀请同学当观众，模拟真实环境

### Q3：我们的项目很简单，没有亮点怎么办？
A3：每个项目都有亮点。CampusFlow 的亮点可能是：
- 极简技术栈（证明不需要 Spring 也能做 Web 开发）
- 测试驱动（80% 覆盖率）
- 文档完善（OpenAPI 规范、ADR 记录）
- 自动化部署（CI/CD 流水线）
- AI 协作经验（AI 生成 + 人工审查）

选择 3-4 个，详细讲。

### Q4：被问到不会的问题怎么办？
A4：诚实回答。"这个问题我们确实没有考虑过，感谢您的建议，我们会记录下来，在下一个版本中改进。"评委不会因为你答不上来而扣分，但会因为胡编乱造而扣分。
