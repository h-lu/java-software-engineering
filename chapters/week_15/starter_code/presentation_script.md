# CampusFlow 演示脚本

总时长：待办： 8-10 分钟

## 开场（1 分钟）

- 待办： 团队/项目一句话介绍。
- 待办： 真实问题背景，说明为什么需要 CampusFlow。

## 主体（约 7 分钟）

### 1. 问题背景

- 待办： 目标用户是谁？
- 待办： 现有方案哪里不够好？

### 2. 技术选型与架构

- 待办： 为什么是 Java + Javalin + SQLite？
- 待办： 用一张图说明核心分层。

### 3. 功能演示

- 待办： 选择 3-4 个最能体现价值的操作。
- 待办： 写出备用演示路径，例如录屏或本地环境。

### 4. 工程实践

- 待办： 测试、文档、部署、质量门禁中最有说服力的证据。

### 5. 踩坑与成长

- 待办： Bug Bash 或 AI 协作中的真实案例。

## 结尾（2 分钟）

- 待办： 总结 CampusFlow 的价值。
- 待办： 下一步计划。
- 待办： 引导评委提问。
