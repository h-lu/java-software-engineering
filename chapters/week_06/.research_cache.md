## AI 小专栏 #2: 为什么 Python 社区偏爱 EAFP

### 搜索词: "Python EAFP philosophy LBYL vs EAFP best practices 2026"

**核心理念**：
- **EAFP** (Easier to Ask Forgiveness than Permission) = "先尝试，再道歉"
- **LBYL** (Look Before You Leap) = "先看再跳"

**关键引用**：
- Python 之禅：**"Errors should never pass silently. Unless explicitly silenced."**
- EAFP 在操作通常成功时**更快**
- LBYL 使代码意图更明确

**使用指南**（来自社区最佳实践）：
- **EAFP**：处理意外/异常事件
- **LBYL**：基于预期的、有效的程序状态进行逻辑判断
