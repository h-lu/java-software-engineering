## AI 小专栏 #1: AI 生成测试代码的边界

### 搜索词: "GitHub Copilot test generation quality 2025"
- 数据点: 学术研究显示，GitHub Copilot 生成的测试代码有效性率达 91.5%（能编译运行），但正确率仅 28.7%（真正验证预期行为）。
- 来源: ACM 研究论文，引用于 https://www.augmentcode.com/tools/github-copilot-vs-qodo
- 引用: "Validity Rate: 91.5% - Tests compile and run without errors; Correctness Rate: 28.7% - Tests actually validate intended behavior correctly"

- 数据点: 2025年6月更新后，GitHub Copilot 消除了语法错误和幻觉方法，所有生成的代码都能编译通过。
- 来源: https://www.c-sharpcorner.com/article/github-copilot-huge-quality-advancement-in-3-months-june-2025/

- 数据点: AI 生成的测试覆盖率达 75%，高于人工编写的 60%，但复杂业务逻辑和边界情况仍需人工干预。
- 来源: https://computerfraudsecurity.com/index.php/journal/article/view/784

### 搜索词: "AI testing code review human oversight 2025"
- 数据点: 新加坡政府技术报告指出，使用 AI 编码助手后，测试覆盖率和代码质量在个人和团队层面都有积极提升，编码速度提高 21-28%。
- 来源: https://docs.developer.tech.gov.sg/docs/ai-coding-assistants/training/files/update-on-ship-hats-coding-assistant-programme.pdf

## AI 小专栏 #2: 参数化测试最佳实践

### 搜索词: "JUnit 5 parameterized test best practices 2025"
- 数据点: JUnit 5.11.x 是当前稳定版本（2026年1月发布），JUnit 6.0.0-RC3 处于早期候选阶段。
- 来源: https://mvnrepository.com/artifact/org.junit.jupiter/junit-jupiter-api/6.0.0-RC3/used-by

- 最佳实践: 使用 @CsvSource 处理简单表格数据，@ValueSource 处理单参数，@MethodSource 仅在需要逻辑构建测试数据时使用。
- 来源: https://ankurm.com/the-complete-guide-to-junit-5-parameterizedtest-write-smarter-faster-and-cleaner-java-tests/

- 最佳实践: 保持数据源"纯粹"——数据提供者中不应包含逻辑，只应返回原始数据。
- 来源: https://www.baeldung.com/parameterized-tests-junit-5

- 最佳实践: 使用描述性测试名称，如 `@ParameterizedTest(name = "{index} => Check if {0} results in {1}")`
- 来源: https://coderpad.io/blog/development/writing-a-parameterized-test-in-junit-with-examples/
