## AI 小专栏 #2: AI 时代的数据库设计决策

### 搜索词: "AI database schema design 2025 2026"

#### AI 驱动 Schema 演进
- 数据点：AI 驱动的 schema 演进可减少 95% 的 schema 相关事故，适应速度提升 10-40 倍
- 来源：https://dzone.com/articles/ai-driven-schema-evolution-adaptive-pipelines
- 来源：https://kinde.com/learn/ai-for-software-engineering/using-ai-for-your-database/ai-powered-database-schema-evolution-from-legacy-to-modern-with-zero-downtime/

#### Expand/Contract 模式（最可靠）
1. Expand：向后兼容地引入新 schema
2. Backfill：为历史数据填充新字段
3. Switch：逐步更新写入/读取（金丝雀/百分比发布）
4. Contract：确认无依赖后移除旧字段

- 来源：https://koder.ai/blog/schema-changes-migrations-evolution-ai-built-systems

#### Agent-AI 系统 Schema 设计原则
- 仅添加可选字段，绝不删除/重命名现有字段
- 通过配置而非代码控制 schema 演进
- 将 prompts 视为 API 进行版本控制
- 来源：https://www.linkedin.com/posts/prakash-kumar-00798221b_agentai-schemaevolution-structuredoutputs-activity-7379556209483218945-Ejbh

| Delta Lake | Schema 演进和强制执行 |
