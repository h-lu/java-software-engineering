## 决策摘要

### ADR-001：领域模型设计

**决策**：采用贫血模型（Anemic Domain Model）
- Task 作为数据实体，业务逻辑在 Service 层
- 优点：简单、易理解、适合小团队
- 缺点：业务逻辑分散，项目大时难以维护

**背景**：Week 02 时团队规模小（2 人），项目复杂度低。

**后果**：如果项目演变成大型系统，需考虑演进为充血模型（Rich Domain Model）。
