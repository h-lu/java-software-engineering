# ADR-001: 领域模型设计

## 状态
已采纳

## 背景
Week 02 设计 CampusFlow 领域模型时，需要决定使用贫血模型还是充血模型。

## 决策
采用贫血模型（Anemic Domain Model）：
- Task 作为数据实体，只包含 getter/setter
- 业务逻辑在 Service 层

## 理由
- 团队规模小（2 人），项目复杂度低
- 贫血模型简单、易理解
- 适合快速迭代

## 后果
- 优点：简单、直观
- 缺点：业务逻辑分散，项目大时难以维护

## 未来演进
如果项目演变成大型系统，需考虑演进为充血模型（Rich Domain Model）。
