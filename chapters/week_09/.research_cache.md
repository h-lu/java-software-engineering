## AI 小专栏 #1: AI 辅助 API 设计

### 搜索词: "AI REST API design 2025 2026"

**关键发现：**
- LLM 现在可以根据自然语言描述生成 OpenAPI 规范
- 研究表明 LLM 在代码坏味道检测上的精确度只有 0.79，召回率仅 0.41
- 企业级实践强调"原子化转换"——每次重构改动控制在 200 行以内

**AI 辅助 API 设计的最佳实践：**
1. **语义清晰性**: 使用自描述的字段名，如 `temperature_celsius` 而非 `temp`
2. **丰富元数据**: 包含单位、数据类型和关系，提高 LLM 理解能力
3. **上下文感知设计**: 支持会话 ID、对话历史和多轮交互的状态管理

来源: https://www.gravitee.io/blog/designing-apis-for-llm-apps

### REST API 端点命名最佳实践（2025 标准）

| 实践 | 示例 | 理由 |
|------|------|------|
| 使用名词，而非动词 | `/users` 而非 `/getUsers` | HTTP 方法已表示动作 |
| 集合用复数名词 | `/products` 而非 `/product` | 一致性和清晰度 |
| 小写连字符 | `/user-profiles` | 更好的可读性，避免大小写问题 |
| 层次结构 | `/users/{id}/orders` | 清晰的资源关系 |
| 无尾部斜杠 | `/devices` 而非 `/devices/` | 更简洁的 URI |
| 路径中包含版本 | `/v1/users` | 清晰的 API 演进 |

来源: https://restfulapi.net/resource-naming/

## AI 小专栏 #2: AI 生成 API 文档

### 搜索词: "OpenAPI AI generation 2025 2026"

**顶级 AI 驱动 API 文档工具（2025-2026）：**

1. **APItoolkit** - AI 原生领导者
   - 从实时生产流量自动生成 OpenAPI/Swagger 文档
   - 分析真实 API 请求以检测结构、字段和格式
   - 自动识别新/更新的字段并提示文档更新

2. **Mintlify** - AI 优先文档
   - 专为 AI 智能体消费 API 而设计
   - 专注于 AI 智能体可有效解析的结构化内容
   - 帮助防止 AI 智能体进行未授权或过度的 API 调用

3. **Apidog** - 新兴 2025-2026
   - AI 功能：自动生成测试用例、智能模拟数据、AI 模式描述
   - 支持 MCP（模型上下文协议），可与 Cursor 等 AI 编码助手集成
   - 设计优先方法，零配置模拟服务器

### 2025-2026 关键趋势

| 趋势 | 影响 |
|------|------|
| AI 智能体消费 | 51% 的开发者担心 AI 智能体进行未授权 API 调用 |
| 实时流量分析 | APItoolkit 等工具从实际 API 使用派生规范，而非手动定义 |
| MCP（模型上下文协议） | 与 AI 编码助手（Cursor、Claude Code 等）集成 |
| 设计优先方法 | 通过从活跃规范生成文档防止文档漂移 |
| 自动生成测试 | AI 直接从 OpenAPI 规范生成测试 |
